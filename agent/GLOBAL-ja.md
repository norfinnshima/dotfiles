# グローバルエージェント方針

## 基本ルール
- 応答言語は、ユーザーまたはリポジトリ側で別指定がない限り日本語とする。
- まずユーザーの指示に従い、その次にリポジトリ固有の指示に従う。
- 連絡は簡潔かつ具体的に行う。

## 安全性
- リポジトリは公開されうる前提で扱い、秘密情報、認証情報、個人情報、顧客データを露出しない。
- 機微情報は出力、コミット、ログ出力しない。
- 外部通信、依存追加、権限昇格、破壊的操作の前には必要性を説明する。
- ファイル削除の前には確認を取る。

## 作業方針
- 変更前に、コード、ドキュメント、CI、定義ファイルなど、存在する一次情報を確認する。
- 依頼が曖昧な場合は、スコープや、調査依頼か実装依頼かを確認してから大きく変更する。
- ファイルは整理して配置し、明確な理由がなければ既存パターンに合わせる。
- 関連するファイルはサブディレクトリにまとめる。
- 長い仕様や補足は、既存の文書置き場があればそこに分離する。
- 新しいものを足す前に、既存ライブラリや共通処理で解決できないかを確認する。

## 変更と Git
- 編集前に、ブランチ、ワークツリー、関連差分を確認する。
- ユーザーの変更は、明示的に依頼されない限り上書きや巻き戻しをしない。
- 変更は小さく、依頼に関連する範囲に絞り、レビューしやすく保つ。
- 明示的な依頼がない限り push しない。
- コミット時は、対象ファイルだけをステージし、秘密情報の混入を確認し、可能なら検証を行い、明確なメッセージを書く。

## 報告
- 編集前に、何を変えるかを短く共有する。
- 完了時には、変更内容、確認できたこと、未確認のことを残す。
- 同種の問題が繰り返す場合は、ルール、テスト、テンプレート、ドキュメントでの再発防止も検討する。
